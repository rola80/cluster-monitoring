# CLAUDE.md — 프로젝트 컨텍스트 (Claude Code용)

> 이 파일은 작업 연속성을 위한 메모리다. 새 세션은 이 파일을 먼저 읽고 시작한다.

## 1. 한 줄 목적
창업·벤처 녹색융합클러스터 **입주 신청서류(구비서류)를 받아 신청요건 적합 여부를 자동 검토**하고,
근거가 달린 판정 리포트(xlsx)를 생성하는 도구. (선정평가·연장평가의 서류적합 검토 자동화)

## 2. 도메인 컨텍스트
- 판정 근거 문서: **모집공고(제2025-004호), 운영규정(제2·9조), 입주기업 관리지침**.
- 한 기업이 약 **12~30종**의 구비서류를 제출(공통서류 1-1~1-9, 가점증빙 2-1~2-20).
- 핵심 자격: **입주신청일 기준 창업 7년 미경과 중소기업**(법인=등기부 회사성립연월일, 개인=사업자등록증 개업연월일) 또는 벤처기업.
- 제외: 운영규정 제9조 미해당, 국세·지방세 체납/부도/파산 등, 허위·부정 제출.
- 가점: 녹색채권 3점, 사회적기업 2점, 인증류·경진/정부지원·우수환경산업체·특허 각 2점/건, 국가R&D 제재 −5점, **합산 최대 5점**.

## 3. 입력 데이터의 현실 (중요)
- 구비서류는 전부 **PDF**, 그러나 **혼합형**: 전자문서(텍스트 레이어 있음)와 **스캔 이미지(텍스트 없음)**가 섞임.
  - 스캔본 예: 입주신청서, 개인정보동의서, 국세·지방세 납세증명서, (일부) 법인등기부등본·인감증명서.
- 일부 문서는 매우 김(매출증빙 200+페이지).
- 중첩 zip + 비밀번호(샘플 비번 `260529`). 한글 정부 양식이라 라벨에 **글자 사이 공백**이 흔함("개 업 연 월 일").

## 4. 판단기준 ↔ 코드
판단기준은 `rules.yaml`(규칙표)에 있고, 각 기준의 `check` 값은 `pipeline/rules_engine.py`의 동명 함수와 1:1.
판정값은 **적합 / 부적합 / 확인필요 / 해당없음**, 모든 결과에 `evidence`(근거)를 남긴다.
**설계 원칙(반드시 준수):**
- 자동 거절 금지 — 추출 신뢰도 낮거나 스캔 미해독이면 `부적합`이 아니라 `확인필요`.
- 최종 판정은 LLM이 아니라 결정형 Python 규칙으로(감사 가능). LLM은 필드 추출 폴백에만.
- 가점·유효기간·녹색산업 분야 적합성은 잠정 처리 후 사람이 최종 확인.

## 5. 아키텍처 / 파일
```
cluster_screening/
  app.py            Streamlit UI
  cli.py            CLI (python -m cluster_screening.cli ...)
  config.py         설정/토글(OCR·LLM·zip비번; 환경변수로 제어)
  rules.yaml        판단기준 규칙표
  pipeline/
    ingest.py        (중첩) zip 해제·PDF 수집 (pyzipper로 AES 대응)
    classify.py      서류 분류 (파일명 힌트 가중치 20 + 본문 키워드)
    extract_text.py  하이브리드 추출: 텍스트레이어(pdfplumber) → 없으면 OCR(pytesseract)
    extract_fields.py 앵커/정규식 추출(+normalize_name, 공백허용 라벨) + LLM 폴백
    rules_engine.py  룰 판정 + evaluate_bonus + 종합판정
    report.py        리포트(xlsx): 종합판정/판단기준별/가점/처리내역 4시트
```
데이터 흐름: `수집 → 분류 → 텍스트추출 → 필드추출 → 룰판정 → 리포트`.

## 6. 현재 상태 (2026-06 기준)
- **에이비알 실제 구비서류 12종으로 E2E 검증 완료 → 종합판정 "적합".**
  - 창업 7년: 개업연월일 2021-03-03 vs 신청일 2026-03-16 = 5.0년차(적합).
  - 체납 없음(납세증명), 서류 간 사업자번호 일치, 필수 7/7 완비.
- 텍스트 레이어 추출 + 룰엔진은 검증됨. **한글 OCR과 OpenAI LLM 폴백은 미검증**
  (개발 샌드박스에 tesseract `kor` 언어팩·OpenAI 키가 없었음). 사용자 로컬에서 활성화 필요.

## 7. 검증에서 얻은 교훈 (회귀 방지)
- 정부서류 라벨은 글자 사이 공백 빈번 → `extract_fields._label_pos`로 공백 허용 매칭.
- **주주명부의 "사업자등록번호"는 회사가 아니라 주주(투자조합)의 번호** → 일치성 검사는
  신뢰서류(사업자등록증·입주신청서·납세증명서·재무제표)로만 한정. 주주명부에서 식별필드 추출 안 함.
- 사업계획서는 본문에 '인증·특허·수상' 단어가 많아 가점서류로 오분류됨 → **파일명 힌트가 본문을 압도**(가중치 20).
- 법인등기부가 스캔이라 회사성립연월일 미추출 시 → 사업자등록증 개업연월일로 폴백.

## 8. 실행
```bash
pip install -r requirements.txt
# 한글 OCR(권장):  apt-get install tesseract-ocr tesseract-ocr-kor poppler-utils
streamlit run cluster_screening/app.py            # UI
python -m cluster_screening.cli <zip|폴더|pdf> --name 기업명 --apply 2026-03-16 --pw 260529
```
환경변수: `ENABLE_OCR`, `OCR_LANG`(기본 kor+eng), `OPENAI_API_KEY`(설정 시 LLM 폴백 ON), `LLM_MODEL`(gpt-4.1-mini), `ZIP_PASSWORD`.

## 10. 컨벤션
- 코드 주석·사용자 출력은 한국어 유지. 모듈 간 상대 임포트(`from .. import config`).
- 새 판단기준 추가 시: `rules.yaml`에 항목 + `rules_engine.py`에 동명 `check_*` 함수.
